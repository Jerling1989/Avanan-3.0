Invisible to hackers

Avanan does not require changes to MX records, and sits behind existing security. This makes it virtually invisible to hackers.

Mail Exchange (MX) records are Domain Name Server (DNS) records which are necessary for the delivery of email. Traditional solutions for email security required changes to these files, which by design need to be publicly visible, so emails can get to their intended addresses. Changing those records, however, also tips off hackers to which security protocols and tools are in use. Designing attacks like Business Email Compromise (BEC) for specific intended targets, the MX record details give malicious actors the roadmap of how to bypass security.

With Avanan, the deployment model enables in-line scanning that works with built-in email security and preserves mail flow rules.
